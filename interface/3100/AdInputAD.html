AdInputAD

Description

The AdInputAD function performs one-shot sampling.
The AdInputAD function uses only the analog-to-digital input function, which differs from the normal continuous sampling input by AdStartSampling function.

Syntax

C 

INT AdInputAD(
HANDLE  hDeviceHandle,
   ULONG  ulCh,
ULONG  ulSingleDiff,
PADSMPLCHREQ  lpAdSmplChReq,
LPVOID  lpData
);

Visual Basic 

Declare Function AdInputAD "FbiAd.DLL" ( _
   ByVal  hDeviceHandle As Long, _
ByVal  ulCh   As Long, _
ByVal  ulSingleDiff As Long, _
ByRef  lpAdSmplChReq As ADSMPLCHREQ, _
ByRef  lpData   As Any _

) As Long

Delphi 

function AdInputAD(
hDeviceHandle : THandle;
ulCh   : Cardinal;
ulSingleDiff : Cardinal;
var lpAdSmplChReq : ADSMPLCHREQ;
lpData   : Pointer

) : DWORD; stdcall; external 'FbiAd.DLL';

Parameters

hDeviceHandle Specifies the device handle obtained by the AdOpen function.
   
ulCh Specifies the number of channels to acquire samples. The settable range is 1 through the number of channels that the boards/cards provide.
   
ulSingleDiff Specifies an input configuration by using the following codes.
Code Description
AD_INPUT_SINGLE Single-ended
AD_INPUT_DIFF Differential
   
lpAdSmplChReq Points to an array of the ADSMPLCHREQ structure containing sampling conditions.
   
lpData Points to a variable to receive samples and stores the analog-to-digital input data. 

Return Value

The AdInputAD function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

  • The ulCh parameter contains the number of channels to sample. For example, if you want to sample channel 1, channel 3, channel 5, and channel 7, specify 4 to ulCh.
  • This function stores the channel number to input to ulChNo of the ADSMPLCHREQ function.
  • This function is not available while the continuous sampling is running.
  • When multiple channels are specified in the following boards/cards, the scan clock differs from when the channel is fixed. Change the setting of scan clock by the AdBmSetSamplingConfig function and change the AD conversion timing.
    PCI-321516
    LPC-321116, LPC-321216, LPC-361116, LPC-361216
    PEX-321116, PEX-321216, PEX-361116, PEX-361216
    CSI/CBI-320412, CBI-320412TR, CBI-320412TK, CBI-320412TL, CSI/CBI-320416, CBI-320416TR, CBI-320416TK, CBI-320416TL, 
    CSI/CBI-360112, CBI-360112TR, CBI-360112TK, CBI-360112TL, CSI/CBI-360116, CBI-360116TR, CBI-360116TK, CBI-360116TL
  • For the bus master boards/cards (except for the PCI/CPZ-3525), the timing of AD conversion depends on the sampling frequency configured by the AdSetSamplingConfig function. The time required for the function is the time configured by the scan clock times the number of channel.
  • Data unit depends on the resolution of the board/card. Please refer to "Data format," and the following table.
    Resolution Data Unit (byte) Language
    C Visual Basic Delphi
    8 bits 1 BYTE Byte Byte
    12 bits 2 WORD Integer Word
    16 bits 2 WORD Integer Word
    24 bits 4 DWORD Long Dword

Examples

C

WORD pData[4];
ADSMPLCHREQ SmplChReq[4];
SmplChReq[0].ulChNo = 1;
SmplChReq[0].ulRange = AD_5V;
SmplChReq[1].ulChNo = 3;
SmplChReq[1].ulRange = AD_5V;
SmplChReq[2].ulChNo = 5;
SmplChReq[2].ulRange = AD_5V;
SmplChReq[3].ulChNo = 7;
SmplChReq[3].ulRange = AD_5V;
nRet = AdInputAD( hDeviceHandle, 4, AD_INPUT_SINGLE, &SmplChReq[0], pData );

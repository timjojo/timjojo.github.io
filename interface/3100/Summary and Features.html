1. Summary

The GPF-3100 software controls Interface analog input boards/cards from your application running on Windows. Application software should link a provided dynamic link library (DLL) and control the analog input boards/cards through the application programming interface (API).

2. Features

  • The GPF-3100 is a hierarchy architecture driver and standardized for Interface analog input boards/cards. You don't have to consider the board/card type except board/card specific features.
  • The GPF-3100 supports up to 230 samples by using the AdStartFileSampling function. When using the AdStartSampling function, the maximum number of samples that can be acquired at a time depends on the amount of memory installed on your computer.
  • The dynamic link library is available to control the Interface analog input boards/cards under Windows 7, Embedded Standard 7, Embedded Standard 2009, XP Embedded, Server 2008, Windows Vista, Windows XP, Windows Server 2003, Windows 2000, Windows NT 4.0, Windows Me, Windows 98, and Windows 95.
  • The AdDataConv and AdReadFile functions allow you to convert sampling data.
  • Sampling can synchronously start and stop with triggers.
  • The data before or after an event can be captured by using trigger delay capabilities.
  • Input range selection and offset/gain calibration are software programmable. (Some boards/cards don't support them.)
  • The Help system explains how to use the functions. Refer to it as you develop your programs.

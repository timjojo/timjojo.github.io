AdOpen

Description

The AdOpen function opens an analog-to-digital input board/card and enables to access to the board/card. To open the board/card, specify a device name of the board/card, FBIADx (x = 1 through 255), automatically specified by the device driver. 

If you use two or more PCI/CompactPCI board/card, check the device names beforehand.  In Windows 7/Server 2008/Vista/XP/Server 2003/2000/Me/98/95, Interface analog-to-digital input board names, RSW1 values, and device names are listed under FbiPciAd in the Device Manager. You can also check these items by using the diagnostic program (DIAGAD.EXE), calibration program (ADCALIB.EXE), or waveform viewer (ADWAVESMP.EXE). The list will be displayed while you use them. 

In Windows NT 4.0, use the diagnostic program (DIAGAD.EXE), calibration program (ADCALIB.EXE), or waveform viewer (ADWAVESMP.EXE) to check these items.

If you change the system configuration (the number of boards/cards or slot position, and so on), assigned device name may also be changed. If you often change the system configuration, we suggest you write your application to be capable of identifying this and alter the device name at run-time.

The CardBus cards have no rotary switch, and the software considers their board number is 0.

Syntax

C (86 bits)

HANDLE AdOpen(
   LPCTSTR   lpszName  
);

C (64 bits)

HANDLE AdOpen(
   LPCTSTR   lpszName  
);

Visual Basic

Declare Function AdOpen Lib "FbiAd.DLL" ( _
   ByVal   lpszName As String _
) As Long 

Delphi

function AdOpen(
   lpszName : Pointer
):Thandle; stdcall; external 'FbiAd.DLL';

Parameter

lpszName  Specifies the device name to open. This device name is from FBIAD1 through FBIAD255.

Return Value

The AdOpen function returns a valid handle when the process is successfully completed. Otherwise, this function returns INVALID_HANDLE_VALUE(FFFFFFFFh) if the process is failed.

Examples

C (32-bit compiler)

HANDLE hDeviceHandle;
hDeviceHandle = AdOpen("FBIAD1");

C (32-bit compiler UNICODE)

HANDLE hDeviceHandle;
hDeviceHandle = AdOpen((LPCTSTR)"FBIAD1");
HANDLE hDeviceHandle;
char* lpszName = "FBIAD1";
hDeviceHandle = AdOpen((LPCTSTR)lpszName);
HANDLE hDeviceHandle;
LPCWSTR UnicodeName = L"FBIAD1";
char DeviceName[256];
int len;
len = WideCharToMultiByte(CP_ACP, 0, UnicodeName, -1, NULL, 0, NULL, NULL);
WideCharToMultiByte(CP_ACP, 0, UnicodeName, -1, DeviceName, len, NULL, NULL);
hDeviceHandle = AdOpen((LPCTSTR)DeviceName);

C (64-bit compiler)

HANDLE hDeviceHandle;
hDeviceHandle = AdOpen("FBIAD1");

Visual Basic 

Dim lpszName As String
Dim hDeviceHandle As Long
lpszName = "FBIAD1" & Chr( 0 )

hDeviceHandle = AdOpen(lpszName)

Delphi

var
   lpszName: array[0..15] of Char;
     
StrCopy( lpszName, 'FBIAD1'); & Chr( 0 )
hDeviceHandle := AdOpen( @lpszName[0] );
Open the board/card whose device name is FBIAD1 and receive a device handle in hDeviceHandle.

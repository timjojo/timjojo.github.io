AdGetBoardConfig

Description

The AdGetBoardConfig function retrieves an event source called on the board.

Syntax

C 

INT AdGetBoardConfig(
HANDLE hDeviceHandle,
   ULONG  *ulAdSmplEventFactor
);

Visual Basic 

Declare Function AdGetBoardConfig Lib _ "FbiAd.DLL" ( _
   ByVal  hDeviceHandle   As Long, _
ByRef ulAdSmplEventFactor  As Long _
) As Long      

Delphi

function AdGetBoardConfig(
   hDeviceHandle : THandle;
   var ulAdSmplEventFactor : Cardinal
) : DWORD; stdcall; external 'FbiAd.DLL';

Parameters

hDeviceHandle Specifies the device handle obtained by the AdOpen function.
  
ulAdSmplEventFactor Points to a variable to receive an event source of the sampling.
Code Value Description
AD_EVENT_SMPLNUM 1 The specified number of samples has been acquired.
AD_EVENT_STOP_TRIGGER 2 The sampling has been stopped because a trigger is asserted.
AD_EVENT_STOP_FUNCTION 3 The sampling has been stopped by software.
AD_EVENT_STOP_TIMEOUT 4 The sampling has been stopped because a time-out interval elapsed.
AD_EVENT_STOP_SAMPLING 5 The sampling is stopped.
AD_EVENT_STOP_SCER 6 The sampling has been stopped because the sampling pacer clock error occurs.
AD_EVENT_STOP_ORER 7 The sampling has been stopped because the overrun error occurs.
AD_EVENT_SCER 8 The sampling pacer clock error occurs.
AD_EVENT_ORER 9 The overrun error occurs.
AD_EVENT_STOP_LV_1 10 The channel 1 sampling is stopped. (Only for the PCI-3179 and CTP/CPZ-3179)
AD_EVENT_STOP_LV_2 11 The channel 2 sampling is stopped. (Only for the PCI-3179 and CTP/CPZ-3179)
AD_EVENT_STOP_LV_3 12 The channel 3 sampling is stopped. (Only for the PCI-3179 and CTP/CPZ-3179)
AD_EVENT_STOP_LV_4 13 The channel 4 sampling is stopped. (Only for the PCI-3179 and CTP/CPZ-3179)
AD_EVENT_RANGE 14 The AD conversion value reached the full-scale range.
AD_EVENT_STOP_RANGE 15 The sampling is stopped by the full-scale range detection.
AD_EVENT_OVPM 16 Automatic range-switching capability at overvoltage input occurs.
AD_EVENT_STOP_OVPM 17 The sampling is stopped by automatic range-switching capability at overvoltage input.

Return Value

The AdGetBoardConfig function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

  • A sampling clock error occurs when the next sampling clock signal inputs during channel scanning. If the sampling clock error is detected, the board continues or stops the sampling depending on the setting.
  • Overrun errors occur when data is lost because the DMA buffer becomes full during sampling. You can set stop/continue sampling by using the AdBmSetSamplingConfig function against overrun error. The number of samples differs from expected number if you set continue sampling.

Target Products

Refer to "Corresponding Identifiers for Each Model". 

Examples

C 

ULONG ulAdSmplEventFactor;
nRet = AdGetBoardConfig( hDeviceHandle, &ulAdSmplEventFactor );

Visual Basic 

Dim ulAdSmplEventFactor As Long
nRet = AdGetBoardConfig( hDeviceHandle, _ ulAdSmplEventFactor )

Delphi

var ulAdSmplEventFactor: Cardinal;
nRet := AdGetBoardConfig( hDeviceHandle, ulAdSmplEventFactor );
Retrieve an event source called on the board specified by hDeviceHandle.

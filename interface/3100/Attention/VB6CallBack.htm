Programming Constraints in Visual Basic 6.0
The variations of Visual Basic 6.0 including, but not limited to, Leaning Edition, Professional Edition, Enterprise Edition, and Service Pack, do not support the callback routine capability. If your program uses the callback routine in Visual Basic 6.0, an application error may occur. In that case, the system will display a message containing words or phrases such as "The memory could not be 'written,''' "Invalid page fault," or "Access violation."

In Visual Basic 6.0, use appropriate notification methods, such as the event signaling, message passing, and/or polling, instead of the callback routine. 

The GPF-3100 supports the event signaling as the notification method. Refer to sample programs about how to use the event signaling.

AdGetSamplingData

Description

The AdGetSamplingData function retrieves sampling data from the board/card.

Syntax

C 

INT AdGetSamplingData(
   HANDLE  hDeviceHandle,
PVOID   pSmplData,
ULONG  *ulSmplNum
);

Visual Basic 

Declare Function AdGetSamplingData Lib _ "FbiAd.DLL" ( _
   ByVal  hDeviceHandle   As Long, _
ByRef  pSmplData As Any, _
ByRef  ulSmplNum As Long _
) As Long      

Delphi

function AdGetSamplingData(
   hDeviceHandle : THandle;
pSmplData : Pointer;
var ulSmplNum : Cardinal
) : DWORD; stdcall; external 'FbiAd.DLL';

Parameters

hDeviceHandle Specifies the device handle obtained by the AdOpen function.
 
pSmplData Points to a buffer to receive samples.
 
ulSmplNum  Points to a variable containing the number of samples to be retrieved. Before calling this function, specify the number of samples to be received. After the function returned, it receives the number of samples actually stored into the buffer pointed by pSmplData.

Return Value

The AdGetSamplingData function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

  • This function transfers samples from the internal buffer of the driver software into user supplied buffer.
  • Data unit depends on the resolution of the board/card. Please refer to "Data Format" and the following table.

  • <PCI-3153 with FIFO transfer mode>
    When you used the GPC-3100 Version 1.41-26 or older versions in the PCI-3153 with FIFO transfer mode, the FIFO status information was added to  the upper 3-bit of sampling data. However, the FIFO status information  is not added in the current version. If you need the digital input data and FIFO status, please use the AdFifoGetSamplingData function.

Examples

C 

PVOID pSmplData;
HANDLE hHeap;
ULONG ulSmplNum
ULONG ulSmplChCount;
  
ulSmplNum = 256;
ulSmplChCount = 3;
hHeap = GetProcessHeap();
pSmplData = HeapAlloc( hHeap, HEAP_ZERO_MEMORY, ulSmplNum * ulSmplChCount * sizeof( WORD ) );
  
nRet = AdGetSamplingData( hDeviceHandle, pSmplData, &ulSmplNum);

Visual Basic 

Dim SmplData(2,255) As Integer
Dim ulSmplNum As Long
  
ulSmplNum = 256
  

nRet = AdGetSamplingData( hDeviceHandle, _ SmplData(0,0), ulSmplNum )

Delphi

var

ulSmplNum := 256;

pSmplData := AllocMem(ulSmplNum * ulSmplChCount * SizeOf(Word));
    
nRet := AdGetSamplingData( hDeviceHandle, pSmplData, ulSmplNum );
Retrieve 256 samples (2 byte/sample) per channel for 3 channels on the 12-bit or 16-bit resolution board/card specified by hDeviceHandle. Samples are stored into the buffer pointed by pSmplData. The ulSmplNum parameter contains the number of samples that are actually acquired after the sampling is successfully completed.

AdBmStartSampling

Description

The AdBmStartSampling function starts continuous sampling on the board/card. When using this function, data is directly DMA transferred to the specified buffer.

Syntax

C

INT AdStartSampling(
 HANDLE hDeviceHandle,
 PVOID   pSmplData,
 ULONG ulSize
);

Visual Basic

Declare Function AdStartSampling Lib "FbiAd.DLL" ( _
 ByVal hDeviceHandle As Long, _
 ByRef pSmplData As Any _
  ByVal ulSize As Long _
) As Long

Delphi

function AdStartSampling(
 hDeviceHandle: THandle;
   pSmplData : Pointer;
  ulSize : Cardinal
) : DWORD; stdcall; external 'FbiAd.DLL';


Parameters

DeviceHandle Specifies the device handle obtained by the AdOpen function.
pSmplData Points to the buffer to store sampling data obtained from the board/card.
ulSize Specifies the buffer size for sampling data obtained from the board/card. 


Return Value

The AdBmStartSampling function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

- board/card other than the PCI/CTP/CPZ-3525, LPC/PEX-320724can use only even number of channels.
- Notes for PCI/CTP/CPZ-3525
Normal clock mode: available only in even number channels
Double clock mode: available only in channel 1

Use only channel 1 in the double clock mode.
The actual sampling frequency is two times as the frequency configured by the AdSetSamplingConfig function or AdBmSetSamplingConfig function.
The number of sampling and delay is specified in even number.    
The configurable number is power of two. 
- This function does not support pre-trigger delay.
Therefore, do not set pre-trigger for the delay by  the AdSetSamplingConfig function or AdBmSetSamplingConfig function.
- This function does not support bus master sumpling status.
Therefore, do not set sapmling status using the AdBmSetSamplingConfig function. 
- Configure the sampling condition using the AdSetSamplingConfig function or AdBmSetSamplingConfig function.
This function does not configure the number of channels and samplings.
- Prepare enough size of DMA buffer. The required size for DMA buffer is  obtained by the following formula;
  
the number of samplings * the number of repetition * Data size of one channel * the number of channels

The lack of the buffer size causes overrun error. Make sure to set the enough size of buffer.
- When an event is registered by the AdSetBoardConfig function and sampling is asynchronously processed, the sampling-stop event is occurred after the number of processing (the specified number of sampling * repetition number) is completed.
- This function is processed asynchronously. 
- Use the AdGetStatus function to confirm the number of the completed samplings and sampling status.

The data size of each data depends on the the resolution. Refer to Data Format)
The table blow shows the variable type for each language.

Resolution Size (Byte) C Visual Basic Delphi
8bit 1 BYTE Byte Byte
10bit 2 WORD Integer Word
12bit 2 WORD Integer Word
16bit 2 WORD Integer Word
24bit 4 DWORD Long Dword

Examples

C

PVOID pSmplData;
ULONG ulSmplNum;
ULONG ulRepeatCount;
ULONG ulSize;

ulChCount = 2;
ulSmplNum = 1024;
ulRepeatCount = 600;

ulSize =ulSmplNum * ulChCount * ulRepeatCount * sizeof( DWORD ); 

pSmplData= calloc(ulSize , sizeof(BYTE));

nRet = AdBmStartSampling( hDeviceHandle, pSmplData, ulSize);

Visual Basic

Dim SmplData(1024, 600) As Long

ulSize = 1024 * 600 * 2 * 4

nRet = AdBmStartSampling( hDeviceHandle, SmplData(0,0), ulSize )

Delphi

var
     pSmplData: Pointer;
     ulChCount : Cardinal;
     ulSmplNum: Cardinal;
     ulRepeatCount: Cardinal;
     ulSize: Cardinal;

ulChCount := 2;
ulSmplNum := 1024;
ulRepeatCount:= 600;

ulSize:= ulSmplNum * ulChCount * ulRepeatCount* SizeOf(DWORD);

pSmplData := AllocMem(ulSize);

nRet := AdBmStartSampling( hDeviceHandle, pSmplData, ulSize );

Start the continuous sampling on the board/card specified by hDeviceHandle

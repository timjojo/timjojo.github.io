11. Reference

11.1 Programming Guide

This section explains how to use the provided functions. Sample codes are written in C.

When you use two or more identical boards, remember to set the RSW1 on each board so as not to conflict with any other before the board installation. The CardBus cards have no rotary switch, and the software considers their board number is 0.

11.1.1 Initializing

The AdOpen function initializes the board.

This is a code fragment illustrating how to use the function. Hereafter, all code fragments are formatted in courier new font.       
hDeviceHandle = AdOpen ("FBIAD1")
A board named "FBIAD1" is initialized.
The device driver assigns device names as follows:
    FBIAD1, FBIAD2, ... , FBIAD255
                   
A device handle is returned to hDeviceHandle after the board is successfully initialized. The device handle allows you to reference the board and must be specified when a function is called.

The initialized board must be closed by the AdClose function before the application is terminated.

11.1.2 Event Settings

The AdSetBoardConfig function setups event conditions. When the AdStartSampling function is performed as an overlapped process (an asynchronous process), the event must be used. If the AdSetBoardConfig function isn't used, default values are used for the settings.
hEvent = CreateEvent(0, TRUE, FALSE, NULL);
AdSetBoardConfig(hDeviceHandle, hEvent, CallBackProc,dwUser);

Executing the AdSetBoardConfig function while sampling is running, the configurations are updated immediately.

11.1.3 Sampling Condition Settings

The AdSetSamplingConfig or AdBmSetSamplingConfig function specifies sampling conditions. Default values are used for sampling conditions, if the AdSetSamplingConfig or AdBmSetSamplingConfig function has not been called. (The default values are retrieved by calling the AdGetSamplingConfig or AdBmGetSamplingConfig function after the board is opened.)
                        
AdSmplConfig.ulChCount = 1;
AdSmplConfig.SmplChReq[0].ulChNo = 1;
AdSmplConfig.SmplChReq[0].ulRange = AD_5V;
AdSmplConfig.ulSamplingMode = AD_IO_SAMPLING;
AdSmplConfig.ulSingleDiff = AD_INPUT_SINGLE;
AdSmplConfig.ulSmplNum = 1024;
AdSmplConfig.ulSmplEventNum = 0;
AdSmplConfig.fSmplFreq = 1000.0;
AdSmplConfig.ulTrigPoint = AD_TRIG_START;
AdSmplConfig.ulTrigMode = AD_FREERUN;
AdSmplConfig.lTrigDelay = 0;
AdSmplConfig.ulTrigCh = 1;
AdSmplConfig.fTrigLevel1 = 0.0;
AdSmplConfig.fTrigLevel2 = 0.0;
AdSmplConfig.ulEClkEdge = AD_DOWN_EDGE;
AdSmplConfig.ulATrgPulse = AD_LOW_PULSE;
AdSmplConfig.ulTrigEdge = AD_DOWN_EDGE;
AdSmplConfig.ulTrigDI = 1;
AdSmplConfig.ulFastMode = AD_NORMAL_MODE;

AdSetSamplingConfig( hDeviceHandle , &AdSmplConfig );

 

Executing the AdSetSamplingConfig or AdBmSetSamplingConfig function while sampling is running, specified sampling conditions are valid when next sampling starts.

11.1.4 Starting the Sampling

The AdStartSampling function starts sampling.
                   

AdStartSampling(hDeviceHandle, FLAG_SYNC);

Specifying the process as a non-overlapped process (FLAG_SYNC), it doesn't return until the sampling is completed. Specifying the process as an overlapped process (FLAG_ASYNC), it returns immediately after calling the function.
By using the function such as the CreateEvent or WaitForSingleObject function of Win32 API, you can wait for the event signal. If you want to know how to use the CreateEvent and WaitForSingleObject functions, please refer to Microsoft Win32 API reference.

11.1.5 Terminating the Sampling

The AdStopSampling function can stop sampling.
                      

AdStopSampling(hDeviceHandle);

If you want to stop the sampling that is performed as an overlapped process, call the AdStopSampling function.

11.1.6 Reading Sampling Data

The AdGetSamplingData or AdBmGetSamplingData function can transfer sampling data from the sampling buffer to the user buffer even if the sampling is running.
                     
ulpSmplNum = 1024;
AdGetSamplingData( hDeviceHandle , pSmplData , &ulpSmplNum );

11.1.7 Closing the Board

The AdClose function closes the board and releases the resources.
                  

AdClose(hDeviceHandle);

11.1.8 Overlapped Process (Asynchronous Process)

The following code fragment shows an overlapped process (an asynchronous process) how to start sampling and detect the sampling completion.
                                   
// Create an event object.
hEvent = CreateEvent(0, TRUE, FALSE, NULL);
AdSetBoardConfig( hDeviceHandle , hEvent , NULL , 0 );

// Starts sampling asynchronously.
AdStartSampling( hDeviceHandle , FLAG_ASYNC );

// Wait until the event object is
// signaled.
WaitForSingleObject(hEvent, INFINITE);

// Reset the event.
ResetEvent(hEvent);

// Close the event handle.
CloseHandle(hEvent);

11.1.9 Example for Overlapped Process (Asynchronous Process)
            in Visual Basic

You need to add the following Win32 function declarations into the standard module.

Declare Function CreateEvent Lib "kernel32" Alias "CreateEventA" (ByVal lpEventAttributes As Long, ByVal  ManualReset As Long, ByVal bInitialState As Long, ByVal lpName As String) As Long
Declare Function WaitForSingleObject Lib "kernel32" (ByVal hHandle As Long, ByVal dwMilliseconds As Long) As Long
Declare Function CloseHandleLib "kernel32" (ByVal hObject As Long) As Long
                               
The following steps show pseudo codes.
1.  Create an event object.
hEvent = CreateEvent(0, True, Fale, 0)
               
2.  Register the event handle.
AdSetBoardConfig(hDeviceHandle, hEvent, 0, 0)
              
3.  Start sampling asynchronously.
AdStartSampling(hDeviceHandle, FLAG_ASYNC)
                
4.  Wait until the event object is signaled.
WaitForSingleObject(hEvent, -1)
                       
5.  Reset  the event.
   ResetEvent (hEvent)
                    
6.  Close the event handle.
   AdStartSampling(hDeviceHandle, FLAG_ASYNC)

11.1.10 Examples of the CallBackProc Function in Visual Basic

1.  Declare your callback routine in the standard module.

Sub lpEventProc(ByVal dwUser As Long)
　'Write your code.
End Sub
2.  Configure callback routine.

AdSetBoardConfig(hDeviceHandle,0,AddressOf lpEventProc,0)
When you follow these procedures, the callback routine (lpEventProc) runs when the sampling is stopped. You can decide the function's name of callback routine. What you need to do is passing the address of the routine to the AdSetBoardConfig function.

AdAllocateSamplingBuffer

Description

The AdAllocateSamplingBuffer function allocates a sampling buffer.

Syntax

C 

INT AdAllocateSamplingBuffer(
     HANDLE  hDeviceHandle
);

Visual Basic 

Declare Function AdAllocateSamplingBuffer Lib _ "FbiAd.DLL" ( _
   ByVal  hDeviceHandle As Long _
) As Long       

Delphi

function AdAllocateSamplingBuffer(
   hDeviceHandle : THandle
) : DWORD; stdcall; external 'FbiAd.DLL';

Parameter

hDeviceHandle Specifies the device handle obtained by the AdOpen function.

Return Value

The AdAllocateSamplingBuffer function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

  • This function is not available while the sampling is running.
  • The size of sampling buffer to be allocated is based on the sampling conditions. If the sampling buffer has already been allocated, it will be reallocated.
  • If the sampling buffer was not allocated by using this function, the AdStartSampling function allocates new sampling buffer when it was called. If the sampling buffer has been allocated by using this function, the AdStartSampling function does not allocate new one.

Examples

C 

nRet = AdAllocateSamplingBuffer( hDeviceHandle );
nRet = AdStartSampling( hDeviceHandle,  FLAG_ASYNC );

Visual Basic

nRet = AdAllocateSamplingBuffer( _ hDeviceHandle )
nRet = AdStartSampling( hDeviceHandle, _ FLAG_ASYNC )

Delphi

nRet := AdAllocateSamplingBuffer( hDeviceHandle );
nRet = AdStartSampling( hDeviceHandle, FLAG_ASYNC );
Start continuous sampling after allocating the sampling buffer of the board specified by hDeviceHandle.

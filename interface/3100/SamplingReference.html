11.2 Data Acquisition Programming Technique

11.2.1 Continuous Sampling

First, configure the sampling conditions of the board by using the AdSetSamplingConfig function. Then start the continuous sampling by using the AdStartSampling function. The sampling data can be retrieved by using the AdGetSamplingData function. To configure the sampling conditions of the board with the bus master transfer mode, call the AdBmSetSamplingConfig and AdBmGetSamplingData functions.
         
Example (C)    

HANDLE      hDeviceHandle;
ADSMPLREQ   AdSmplConfig;
ULONG       ulSmplNum;
WORD        wSmplData[1024][2];

hDeviceHandle = AdOpen( "FBIAD1" );

AdSmplConfig.ulChCount = 2;
AdSmplConfig.SmplChReq[0].ulChNo = 1;
AdSmplConfig.SmplChReq[0].ulRange = AD_5V;
AdSmplConfig.SmplChReq[1].ulChNo = 2;
AdSmplConfig.SmplChReq[1].ulRange = AD_5V;
AdSmplConfig.ulSamplingMode = AD_IO_SAMPLING;
AdSmplConfig.ulSingleDiff = AD_INPUT_SINGLE;
AdSmplConfig.ulSmplNum = 1024;
AdSmplConfig.ulSmplEventNum = 0;
AdSmplConfig.fSmplFreq = 1000.0;
AdSmplConfig.ulTrigPoint = AD_TRIG_START;
AdSmplConfig.ulTrigMode = AD_FREERUN;
AdSmplConfig.lTrigDelay = 0;
AdSmplConfig.ulTrigCh = 1;
AdSmplConfig.fTrigLevel1 = 0.0;
AdSmplConfig.fTrigLevel2 = 0.0;
AdSmplConfig.ulEClkEdge = AD_DOWN_EDGE;
AdSmplConfig.ulATrgPulse = AD_LOW_PULSE;
AdSmplConfig.ulTrigEdge = AD_DOWN_EDGE;
AdSmplConfig.ulTrigDI = 1;
AdSmplConfig.ulFastMode = AD_NORMAL_MODE;

// Configure the sampling conditions.
AdSetSamplingConfig( hDeviceHandle, &AdSmplConfig );

// Start the sampling.
AdStartSampling( hDeviceHandle, FLAG_SYNC );

// Retrieve the sampling data.
ulSmplNum = 1024;
AdGetSamplingData( hDeviceHandle, &wSmplData[0][0], &ulSmplNum );
.
.
.

11.2.2 One-Shot Sampling

The software-paced one-shot sampling needs to use the AdInputAD function.
             
Example (C)   

HANDLE      hDeviceHandle;
ADSMPLCHREQ AdSmplChReq[2];
WORD        wData[2];

hDeviceHandle = AdOpen( "FBIAD1" );

AdSmplChReq[0].ulChNo = 1;
AdSmplChReq[0].ulRange = AD_5V;
AdSmplChReq[1].ulChNo = 2;
AdSmplChReq[1].ulRange = AD_5V;

// One-shot sampling
AdInputAD( hDeviceHandle, 2, AD_INPUT_SINGLE, &AdSmplChReq[0], &wData[0]);

11.2.3 Parallel Sampling

In the parallel sampling configuration, as a single master and one or more slave boards exist in the system. The following shows how to start the parallel sampling.
1. Configure the sampling conditions for each board by using the AdSetSamplingConfig function.
2. Call the AdSyncSampling function in the slave boards.
3. Call the AdSyncSampling function in the master board.
Then the parallel sampling starts. The samples can be retrieved by using the AdGetSamplingData function in each board.

When you use boards that support the bus master mode, you can use the AdBmSetSamplingConfig and AdBmGetSamplingData functions.
           
Example (C)   

HANDLE     hDeviceMaster;
HANDLE     hDeviceSlave1;
HANDLE     hDeviceSlave2;
ADSMPLREQ  AdConfigMaster;
ADSMPLREQ  AdConfigSlave1;
ADSMPLREQ  AdConfigSlave2;
WORD       wDataMaster[1024][2];
WORD       wDataSlave1[1024][2];
WORD       wDataSlave2[1024][2];
ULONG      ulSmplNum;

hDeviceMaster = AdOpen( "FBIAD1" );
hDeviceSlave1 = AdOpen( "FBIAD2" );
hDeviceSlave2 = AdOpen( "FBIAD3" );

AdConfigMaster.ulChCount = 2;
AdConfigMaster.SmplChReq[0].ulChNo = 1;
AdConfigMaster.SmplChReq[0].ulRange = AD_5V;
AdConfigMaster.SmplChReq[1].ulChNo = 2;
AdConfigMaster.SmplChReq[1].ulRange = AD_5V;
AdConfigMaster.ulSamplingMode = AD_IO_SAMPLING;
AdConfigMaster.ulSingleDiff = AD_INPUT_SINGLE;
AdConfigMaster.ulSmplNum = 1024;
AdConfigMaster.ulSmplEventNum = 0;
AdConfigMaster.fSmplFreq = 1000.0;
AdConfigMaster.ulTrigPoint = AD_TRIG_START;
AdConfigMaster.ulTrigMode = AD_FREERUN;
AdConfigMaster.lTrigDelay = 0;
AdConfigMaster.ulTrigCh = 1;
AdConfigMaster.fTrigLevel1 = 0.0;
AdConfigMaster.fTrigLevel2 = 0.0;
AdConfigMaster.ulEClkEdge = AD_DOWN_EDGE;
AdConfigMaster.ulATrgPulse = AD_LOW_PULSE;
AdConfigMaster.ulTrigEdge = AD_DOWN_EDGE;
AdConfigMaster.ulTrigDI = 1;
AdConfigMaster.ulFastMode = AD_NORMAL_MODE;

// Configure the sampling conditions
// (master board).
AdSetSamplingConfig( hDeviceMaster, &AdConfigMaster );

AdConfigSlave1.ulChCount = 2;
AdConfigSlave1.SmplChReq[0].ulChNo = 1;
AdConfigSlave1.SmplChReq[0].ulRange = AD_5V;
AdConfigSlave1.SmplChReq[1].ulChNo = 2;
AdConfigSlave1.SmplChReq[1].ulRange = AD_5V;
.
.
.

// Configure the sampling conditions
// (slave board 1).
AdSetSamplingConfig( hDeviceSlave1, &AdConfigSlave1 );

AdConfigSlave2.ulChCount = 2;
AdConfigSlave2.SmplChReq[0].ulChNo = 1;
AdConfigSlave2.SmplChReq[0].ulRange = AD_5V;
AdConfigSlave2.SmplChReq[1].ulChNo = 2;
AdConfigSlave2.SmplChReq[1].ulRange = AD_5V;
.
.
.

// Configure the sampling conditions
// (slave board 2).
AdSetSamplingConfig( hDeviceSlave2, &AdConfigSlave2 );

// Parallel sampling (slave boards)
AdSyncSampling( hDeviceSlave1, AD_SLAVE_MODE );
AdSyncSampling( hDeviceSlave2, AD_SLAVE_MODE );

// Parallel sampling (master board)
AdSyncSampling( hDeviceMaster, AD_MASTER_MODE );
.
.
.

// Retrieve the sampling data (master board).
ulSmplNum = 1024;
AdGetSamplingData( hDeviceMaster, &wDataMaster[0][0], &ulSmplNum );

// Retrieve the sampling data (slave board 1).
ulSmplNum = 1024;
AdGetSamplingData( hDeviceSlave1, &wDataSlave1[0][0], &ulSmplNum );

// Retrieve the sampling data (slave board 2).
ulSmplNum = 1024;
AdGetSamplingData( hDeviceSlave2, &wDataSlave2[0][0], &ulSmplNum );
.
.

Note: You can use the same external signal to synchronize sampling timing on each board. Refer to "External Triggers" and "External Clock," respectively.

11.2.5 External Triggers

The sampling can start at the assertion of an external trigger. Configure the sampling conditions by using the AdSetSamplingConfig function. The AdStartSampling function starts continuous sampling on the board. The samples can be retrieved by using the AdGetSamplingData function.

When you use boards which support the bus master mode, you can use the AdBmSetSamplingConfig and AdBmGetSamplingData functions.
    
Example (C)  

HANDLE     hDeviceHandle;
ADSMPLREQ  AdSmplConfig;
ULONG      ulSmplNum;
WORD       wSmplData[1024][2];

hDeviceHandle = AdOpen( "FBIAD1" );

AdSmplConfig.ulChCount = 2;
AdSmplConfig.SmplChReq[0].ulChNo = 1;
AdSmplConfig.SmplChReq[0].ulRange = AD_5V;
AdSmplConfig.SmplChReq[1].ulChNo = 2;
AdSmplConfig.SmplChReq[1].ulRange = AD_5V;
AdSmplConfig.ulSamplingMode = AD_IO_SAMPLING;
AdSmplConfig.ulSingleDiff = AD_INPUT_SINGLE;
AdSmplConfig.ulSmplNum = 1024;
AdSmplConfig.ulSmplEventNum = 0;
AdSmplConfig.fSmplFreq = 1000.0;
AdSmplConfig.ulTrigPoint = AD_TRIG_START;
// Start sampling by the trigger. 
AdSmplConfig.ulTrigMode = AD_EXTTRG;
// External trigger
AdSmplConfig.lTrigDelay = 0;
AdSmplConfig.ulTrigCh = 1;
AdSmplConfig.fTrigLevel1 = 0.0;
AdSmplConfig.fTrigLevel2 = 0.0;
AdSmplConfig.ulEClkEdge = AD_DOWN_EDGE;
AdSmplConfig.ulATrgPulse = AD_LOW_PULSE;
AdSmplConfig.ulTrigEdge = AD_DOWN_EDGE;
AdSmplConfig.ulTrigDI = 1;
AdSmplConfig.ulFastMode = AD_NORMAL_MODE;

// Configure the sampling conditions.
AdSetSamplingConfig( hDeviceHandle, &AdSmplConfig );

// Wait for an assertion of the external trigger.
// Starts the continuous sampling.
AdStartSampling( hDeviceHandle, FLAG_SYNC );

// Retrieve the sampling data.
ulSmplNum = 1024;
AdGetSamplingData( hDeviceHandle, &wSmplData[0][0], &ulSmplNum );
.
.
.

             
Please refer to "External Trigger Function."

When you use boards which support the bus master mode, you can use the AdBmSetSamplingConfig and AdBmGetSamplingData functions.
    
Example (C)

ADSMPLREQ  AdSmplConfig;
ULONG      ulSmplNum;

hDeviceHandle = AdOpen( "FBIAD1" );

11.2.7 Trigger-Paced Sampling

The AdTriggerSampling function samples a single channel on the board at each trigger asserted on the EXTRG IN pin. This function is only applicable to the boards whose analog input channels are multiplexed into an AD converter and whose data transfer mode is the programmed I/O mode.
                  
Example (C)  

HANDLE     hDeviceHandle;
ULONG      ulSmplNum;
WORD       wSmplData[1024];

hDeviceHandle = AdOpen( "FBIAD1" );

// Samples a single channel on the board at each
// trigger asserted on the EXTRG IN pin.
AdTriggerSampling( hDeviceHandle, 1, AD_5V, AD_INPUT_SINGLE, AD_EXTTRG, AD_DOWN_EDGE, 1024 );

// Retrieve the sampling data.
ulSmplNum = 1024;
AdGetSamplingData( hDeviceHandle, &wSmplData[0], &ulSmplNum );
.
.
.

                       
Please refer to the AdTriggerSampling function and "External Trigger Function."

11.2.8 Sampling for the CSI/CBI-320110, CTP/CPZ-360810, LPC/PEX-320910

For the sampling of theCSI/CBI-320110, CTP/CPZ-360810 and LPC-320910, PEX-320910, the AdMemSetSamplingConfig function configures the sampling conditions and the AdStartSampling function starts the sampling. The AdGetSamplingData function retrieves the sampling data.
  
Example (C)   
HANDLE         hDeviceHandle;
ADMEMSMPLREQ   AdSmplConfig;
ULONG          ulSmplNum;
WORD           wSmplData[1024][2];
hDeviceHandle = AdOpen( "FBIAD1" );

// Configures the sampling conditions.
AdMemGetSamplingConfig( hDeviceHandle, &AdSmplConfig );

AdSmplConfig.ulChCount = 2;
AdSmplConfig.SmplChReq[0].ulChNo = 1;
AdSmplConfig.SmplChReq[0].ulRange = AD_1V;
AdSmplConfig.SmplChReq[1].ulChNo = 2;
AdSmplConfig.SmplChReq[1].ulRange = AD_1V;
AdSmplConfig.fSmplFreq = 1000.0;
AdSmplConfig.ulStopMode = AD_STOP_SOFT;
AdSmplConfig.ulPreTrigDelay = 1024;
AdSmplConfig.ulPostTrigDelay = 0;

// Configures the sampling conditions.
AdMemSetSamplingConfig( hDeviceHandle, &AdSmplConfig );
// Starts the sampling.(asynchronous sampling)
AdStartSampling( hDeviceHandle, FLAG_ASYNC );
  :
// waiting for a trigger. 
  :
// Stops the sampling.
AdStopSampling( hDeviceHandle );
  :
  :
// Retrieve the sampling data.
ulSmplNum = 1024;
AdGetSamplingData( hDeviceHandle, &wSmplData[0][0], &ulSmplNum );

AdLvSetSamplingConfig

Description

The AdLvSetSamplingConfig function configures sampling conditions of the board/card. 

Syntax

C 

INT AdLvSetSamplingConfig(
HANDLE  hDeviceHandle,
ULONG  ulChNo,
ULONG  ulSmplNum,
float fSmplFreq,
ULONG ulRange,
HANDLE hEvent,
LPADCALLBACK  lpCallBackProc,
DWORD  dwUser
);

Visual Basic

   ByVal  hDeviceHandle   As Long, _
ByVal  ulChNo   As Long, _
ByVal  ulSmplNum   As Long, _ 
ByVal  fSmplFreq   As Single, _
ByVal  ulRange As Long, _
ByVal  hEvent   As Long, _
ByVal  lpCallBackProc  As Long, _
ByVal  dwUser   As Long _
) As Long

Parameters

hDeviceHandle

Specifies the device handle obtained by the AdOpen function.
        
ulChNo Specifies an analog input channel. Only one channel can be specified.
    
ulSamplNum Specifies the number of samples to be acquired.
      
fSmplFreq Specifies the sampling rate. The following sampling rates are available:
          50 Hz, 60 Hz, 250 Hz, and 500 Hz.
     
ulRange Specifies the analog input range.

Code

 Value

Description
AD_1P25V 0x00800000 Voltage range: bipolar +/-1.25 V
AD_0P125V 0x00400000 Voltage range: bipolar +/-0.125 V
AD_0P156V 0x02000000 Voltage range: bipolar +/-0.156 V
AD_0P625V 0x01000000 Voltage range: bipolar +/-0.625 V
AD_1P25V_AC 0x04000000 Voltage range: bipolar +/-1.25 V
(with AC coupling)
AD_0P156V_AC 0x10000000 Voltage range: bipolar +/-0.156 V
(with AC coupling)
AD_0P625V_AC 0x08000000 Voltage range: bipolar +/-0.625 V
(with AC coupling)
AD_0_1P25V 0x08000000 Voltage range: unipolar 0 V to +1.25 V
AD_0_0P125V 0x00000080 Voltage range: unipolar 0 V to +0.125 V
AD_0_0P156V 0x00000200 Voltage range: unipolar 0 V to +0.156 V
AD_0_0P625V 0x00000100 Voltage range: unipolar 0 V to +0.625 V
        
hEvent Specifies an event to be signaled when a sampling terminates or the specified number of samples are successfully acquired. If you don't use event notification, specify it as follows:
  • NULL (C)
  • 0 (Visual Basic)
  • nil (Delphi)
lpCallBackProc Specifies an address of user callback routine to be called when the sampling stops. If you do not use a callback routine, specify it as follows:
  • NULL (C)
  • 0 (Visual Basic)
  • nil (Delphi)
dwUser Specifies a user data to be passed to the user callback routine. If you do not use the user callback routine, please specify 0. 

Return Value

The AdLvSetSamplingConfig function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

  • This function is not available while the sampling is running.
  • The sampling conditions configured by this function are applied to the sampling by using the AdLvStartSampling function.
  • If you change sampling conditions of channel by the AdSetSamplingConfig function after configuring the same channel by this function, the range and sampling rate are changed.
  • The following boards/cards are applicable to use this function.
    Model AC Coupling Capability
    PCI-3179 -
    CTP/CPZ-3179 Available

Examples

C 

hEvent = CreateEvent(0, TRUE, FALSE, NULL);
nRet = AdLvSetSamplingConfig( hDeviceHandle, 1, 1000, 50.0, AD_1P25V, hEvent, CallBackProc, 0);

Visual Basic

nRet = AdLvSetSamplingConfig( hDeviceHandle, 1, _ 1000, 50.0, AD_1P25V, hEvent, CallBackProc, 0 )

Delphi

nRet := AdLvSetSamplingConfig( hDeviceHandle, 1, 1000, 50.0, AD_1P25V, hEvent, CallBackProc, 0 );

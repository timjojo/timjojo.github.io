AdTriggerSampling

Description

The AdTriggerSampling function samples a single channel on the board/card at each trigger asserted on the EXTRG IN pin. This function is applicable only to boards/cards whose analog input channels are multiplexed into an AD converter and whose data transfer mode is the programmed I/O mode.

ULONG ulChNo, 
ULONG  ulRange, 
ULONG ulSingleDiff,

ULONG ulTrigEdge,

Parameters

hDeviceHandle Specifies the device handle obtained by the AdOpen function.
   
ulChNo Specifies an analog input channel. Only one channel can be specified.
   
ulRange Specifies the analog input range.
Code Value Description
AD_0_1V 0x00000001 Voltage range: unipolar 0 V to +1 V
AD_0_2P5V 0x00000002 Voltage range: unipolar 0 V to +2.5 V
AD_0_5V 0x00000004 Voltage range: unipolar 0 V to +5 V
AD_0_10V 0x00000008 Voltage range: unipolar 0 V to +10 V
AD_1_5V 0x00000010 Voltage range: unipolar +1 V to +5 V
AD_0_2V 0x00000020 Voltage range: unipolar +1 V to +2 V
AD_0_20mA 0x00001000 Current range: unipolar 0 mA to +20 mA
AD_4_20mA 0x00002000 Current range: unipolar +4 mA to 
+20 mA
AD_1V 0x00010000 Voltage range: bipolar +/-1 V
AD_2P5V 0x00020000 Voltage range: bipolar +/-2.5 V
AD_5V 0x00040000 Voltage range: bipolar +/-5 V
AD_10V 0x00080000 Voltage range: bipolar +/-10 V
    
ulSingleDiff Specifies the input configuration by using the following codes.
Code Value Description
AD_INPUT_SINGLE 1 Single-ended
AD_INPUT_DIFF 2 Differential
  
ulTriggerMode Specifies the trigger condition by using the following codes.
Code Value Description
AD_EXTTRG 2 External trigger (EXTRG IN)
AD_EXTTRG_DI 3 External trigger with a mask using digital input pin IN1 (EXTRG IN, IN1). While a signal on the IN1 pin is high-level, an assertion of the external trigger on the EXTRG IN pin is ignored.
   
ulTrigEdge Specifies a polarity of the external trigger by using the following codes.
Code Value Description
AD_DOWN_EDGE 1 Falling edge
AD_UP_EDGE 2 Rising edge
    
ulSmplNum Specifies the number of samples to be acquired in the range of 1 through 1G.

Return Value

The AdTriggerSampling function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Comments

  • This function is applicable to the following boards/cards.
    PCI-3126, PCI-3133, PCI-3135, PCI-3153, PCI-3155,  PCI-3165, PCI-3166, PCI-3168C, PCI-3170A, PCI-3171A, PCI-3172A, PCI-3173A, PCI-3177C, PCI-3521, PCI-3522A, PCI-3523A
    CTP/CPZ-3126, CTP/CPZ-3133, CTP/CPZ-3135, CTP/CPZ-3165, CTP/CPZ-3167, CTP/CPZ-3168, CTP/CPZ-3170, CTP/CPZ-3171, CTP/CPZ-3172, CTP/CPZ-3173, CTP/CPZ-3177, CTP/CPZ-3521, CTP/CPZ-3522, CTP/CPZ-3523
    CBI-3133A, CBI-3133B, CBI-3134A, CBI-3134B
  • To configure the sampling condition of the board/card, use the AdSetSamplingConfig function.
  • This function is always performed as an overlapped operation.
  • This function can handle only one channel at a time.
  • Event notification and callback routine capability are available when the sampling is completed. To configure the event notification and the callback routine, use the AdSetBoardConfig function.
  • To abort the running sampling, use the AdStopSampling function.
  • To retrieve the data from the buffer, use the AdGetSamplingData function.
  • Use AdStartSampling function in the external clock for the boards/cards that do not correspond to this function. 

Examples

C 

nRet = AdTriggerSampling( hDeviceHandle, 1, AD_5V, AD_INPUT_SINGLE, AD_EXTTRG, AD_DOWN_EDGE, 1024);

Visual Basic 

nRet = AdTriggerSampling( hDeviceHandle, 1, _ AD_5V, AD_INPUT_SINGLE, AD_EXTTRG, _ AD_DOWN_EDGE, 1024 )

Delphi

nRet = AdTriggerSampling ( hDeviceHandle, 1, AD_5V, AD_INPUT_SINGLE, AD_EXTTRG, AD_DOWN_EDGE, 1024 );
Acquire 1024 data from the board/card specified by hDeviceHandle under the following conditions.
Parameter Setting
Channel number 1
Input range +/-5 V
Input configuration Single-ended
Trigger conditions External trigger, falling edge

AdStartFileSampling

Description

The AdStartFileSampling function starts continuous sampling and saves the data into a data file simultaneously.

Syntax

C 

INT AdStartFileSampling(
HANDLE  hDeviceHandle,
LPCTSTR  pszPathName,
   ULONG  ulFileFlag
);

Visual Basic 

Declare Function AdStartFileSampling Lib _ "FbiAd.DLL" ( _
ByVal  hDeviceHandle As Long, _
ByVal  pszPathName   As String, _
   ByVal  ulFileFlag   As Long _
) As Long                

Delphi

function AdStartFileSampling(
hDeviceHandle : THandle;
pszPathName   : Pointer;
   ulFileFlag   : Cardinal
) : DWORD; stdcall; external 'FbiAd.DLL';

Parameters

hDeviceHandle  Specifies the device handle obtained by the AdOpen function.
  
pszPathName Specifies the data file containing the sampling data.
  
ulFileFlag Specifies the format of the data file.
Flag Value Description
FLAG_BIN 1 Binary format
FLAG_CSV 2 CSV format (physical value)

Return Value

The AdStartFileSampling function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns an another code. Please refer to error codes.

Comments

  • This function does not return until the continuous sampling has been completed.
  • The sampling conditions specified by the AdSetSamplingConfig function are used for this sampling.
  • The following trigger conditions can be selected. Triggers not listed below are not available.
         >> No trigger
         >> Start-trigger without trigger-delays
         >> Start-trigger with post-trigger-delays
  • Data unit depends on the resolution of the board. Please refer to "Data format."
  • In this function, the binary and the CSV format are available. For the binary format, please refer to the following table.

    Resolution

    Valid Bit
    8 bits All 8 bits of the 8-bit data
    12 bits Lower 12 bits of the 16-bit data
    16 bits All 16 bits of the 16-bit data
    24 bits Lower 24 bits of the 32-bit data
  • For the CSV format, the data are represented in physical values such as voltage or current. You do not need to consider the resolution of the board. The data are separated from each other by commas.
  • If the data are saved in the CSV format, the sampling may not operate at the specified rate because the data conversion from binary to physical value needs more processing time or overhead.
  • All of the data in the sampling buffer of the previous acquisition are cleared when the next acquisition starts by using this function.
  • If the specified file has already been existed, the file is overwritten.

Examples

C 

nRet = AdStartFileSampling( hDeviceHandle, "DATA.CSV" , FLAG_CSV );

Visual Basic 

Dim szPathName As String
pszFname = "DATA.CSV"& Chr(0)
nRet = AdStartFileSampling( hDeviceHandle, _ szPathFname , FLAG_CSV )

Delphi

var
szPathName : array[0..15] of Char;
  
StrCopy(pszPathName, 'DATA.CSV');
nRet := AdStartFileSampling( hDeviceHandle, @szPathName , FLAG_CSV );
Start continuous sampling on the board specified by hDeviceHandle with simultaneous streaming of the data into "DATA.CSV" in the CSV format.

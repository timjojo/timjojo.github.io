5. Data Transfer Modes

Our analog input board/card provide the following data transfer modes depending on their hardware specifications.
  • Programmed I/O
  • FIFO
  • Memory
  • Bus master

You can select a data transfer mode by using the AdSetSamplingConfig function. The AdBmSetSamplingConfig and function is applicable to the bus master transfer mode.
AdLvSetSamplingConfig function is applicable to the PCI-3179, CTP/CPZ-3179.
AdMemSetSamplingConfig function is applicable to the LPC-320910, PEX-320910, CPZ-360810, CSI/CBI-320110, CBI-320110TK, CBI-320110TL. 

To confirm your board/card specifications, please refer to the AdGetDeviceInfo function.

Analog input boards acquire a signal sample at a time in the programmed I/O mode. In this mode, I/O transfers should be finished between the board registers and computer memories every sampling. All trigger capabilities and trigger delay capabilities are available and there is no restriction to the number of samples.

FIFO memory equipped in analog input boards can continuously acquire samples and store them into the FIFO memory. In this mode, the system overheads are drastically reduced.
The sampling rates will become faster than the programmed I/O mode.

There may be limitations for triggering, trigger delay, event capabilities, and the number of samples depending on the hardware specifications. Refer to "Attentions" for more details.

Buffer memory equipped in analog input boards/cards can continuously acquire samples and store them into the buffer memory. In this mode, the system overheads are drastically reduced. This flat buffer memory is mapped to the system PCI memory space. This mode provides the fastest sampling rate.

There may be limitations for triggering, trigger delay, event capabilities, and the number of samples depending on the hardware specification.

PCI Bus master DMA (direct memory access) allows analog input boards/cards to acquire samples and store them into the main memory of the computer without interventions of CPU. In this mode, the system overheads are drastically reduced. This mode allows you to use other applications smoothly during data transfer.

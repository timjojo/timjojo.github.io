AdCalibration

Description

The AdCalibration function calibrates analog-to-digital input device. 
Some devices do not support electric volume. When a device which does not electric volume is specified, the AdClalibration function returns AD_Error_NOT_SUPPORTED.

Syntax

C 

INT AdCalibration(
   HANDLE  hDeviceHandle,
ULONG   ulAdjustCh,
ULONG  ulRange,
ULONG  ulSingleDiff
);

Visual Basic 

Declare Function AdCalibration Lib "FbiAd.DLL"(
   ByVal  hDeviceHandle  As Long, _
ByVal  ulAdjustCh As Long,
ByVal ulRange  As Long _
ByVal ulSingleDiff  As Long _
) As Long      

Delphi

function AdCalibration(
   hDeviceHandle : THandle;
ulAdjustCh : Cardinal;
ulRange  : Cardinal;
ulSingleDiff : Cardinal;
) : DWORD; stdcall; external 'FbiAd.DLL';

Parameters

hDeviceHandle Specifies the device handle obtained by the AdOpen function.
 
ulAdjustCh Specifies a channel number to calibrate. Always specifies 1 for multiplexer devices.
 
ulRange  Specifies the range to calibrate.
To specify a range, specify either one of the range identifiers below..

ulSingleDiff Specifies the input method.

Identifier

Value

Description
AD_INPUT_SINGLE  1 Single ended input 
AD_INPUT_DIFF 2 Differential input

Return Value

The AdCalibration function returns AD_ERROR_SUCCESS if the process is successfully completed. Otherwise, this function returns another code. Please refer to the error codes.

Examples

C 

nRet = AdCalibration( hDeviceHandle, 1, AD_10V, AD_INPUT_SINGLE );

Visual Basic 

nRet = AdCalibration( hDeviceHandle, 1, AD_10V, AD_INPUT_SINGLE )

Delphi

nRet := AdCalibration( hDeviceHandle, 1, AD_10V, AD_INPUT_SINGLE );
Performing an auto-calibration for channel 1 of  the analog-to-digital input device, +/- 10V range, single ended input specified in  hDevicehandle.

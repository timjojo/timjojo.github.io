7. Trigger Delays

Using the trigger delay capability, actual analog input starts or stops after a trigger is asserted. The GPF-3100 has the following delay capabilities.
     
  • Post-Trigger Delay
    An actual start-point or stop-point is delayed by the time according to the number of post-trigger samples.
  • Pre-Trigger Delay
    An actual start-point precedes the trigger point by the time according to the number of pre-trigger samples. 
The number of pre-trigger samples and the number of post-trigger samples are specified by the AdSetSamplingConfig and AdBmSetSamplingConfig functions.
                            
1. Start-Trigger with Post-Trigger Delay
An actual acquired samples include pre-trigger samples.
                      
2. Start-Trigger with Post-Trigger Delay

           Pre-trigger samples are abandoned. An actual acquired samples don't include the pre-trigger samples.
                    
3. Stop-Trigger with Post-Trigger Delay
    An actual acquired samples include the post-trigger samples.
    
The following figures describes the condition when using the CBI/CSI-320110. The AdMemSetSamplingConfig function configures the number of the pre-trigger and post-trigger samples. The pre-trigger and post-trigger delays are only applicable to the stop-trigger. There is no configuration of the number of samplings. The number of samplings is recognized as the number of samplings between stop-trigger and pre-trigger or post-trigger. Therefore, it is unnecessary to specify the number of samplings.
